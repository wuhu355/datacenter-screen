## 一、技术选型

**组合 2：纯 HTML + ECharts + Python Flask**

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | 纯 HTML + CSS + JS | 单页大屏无需框架，ECharts CDN 引入 |
| 后端 | Python Flask | 课程用 Python，80 行搞定所有 API |
| 数据库 | MySQL 8.0 Docker | 已有，monitor 库 79,275 条数据 |
| 图表 | ECharts 5.x | 国内大屏标配，暗色主题开箱即用 |
| 实时策略 | 时间窗口截取 + 翻牌动画 | 展示最近时段数据，统计卡片数字递增动画 |

## 三、API 设计（10 个接口）

| # | API | 方法 | 说明 |
|---|-----|------|------|
| 1 | `/api/stats` | GET | 基础统计：主机数、在线数、指标数、告警数、健康度 |
| 2 | `/api/room-health` | GET | 5 个机房的 CPU/磁盘/内存综合评分 |
| 3 | `/api/host-risk` | GET | 主机风险排名 TOP5（加权评分） |
| 4 | `/api/disk-top` | GET | 磁盘使用率 TOP5 |
| 5 | `/api/cpu-trend` | GET | CPU 各指标 7 天小时级趋势 |
| 6 | `/api/load-trend` | GET | 系统负载 load1/5/15 趋势 |
| 7 | `/api/disk-io` | GET | 磁盘读写 TOP5（read + write 堆叠） |
| 8 | `/api/memory-top` | GET | 内存使用 TOP5 |
| 9 | `/api/network` | GET | 网络出入流量趋势 |
| 10 | `/api/alerts` | GET | 基于阈值规则生成的告警列表 |

## 四、告警规则

| 规则 | 条件 | 级别 |
|------|------|------|
| CPU 过高 | `cpu_user + cpu_sys` > 70% | 🔴 严重 |
| 磁盘使用率 | util > 80% | ⚠️ 警告 |
| 磁盘濒满 | util > 95% | 🔴 严重 |
| 内存高水位 | mem_used > 75000 MB | ⚠️ 警告 |
| 负载过高 | load1 > 24 | ⚠️ 警告 |
