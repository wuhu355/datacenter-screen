## 📊 二、时间戳解析

### 2.1 原理

`disk_tsar.ts` 和 `pref_tsar.ts` 存储的是 **Unix 毫秒时间戳**（距 1970-01-01 00:00:00 UTC 的毫秒数）。

| 时间戳(毫秒) | 除以1000(秒级) | 人类可读时间 |
|-------------|---------------|-------------|
| `1782835200000` | `1782835200` | 2026-07-01 00:00:00 |
| `1786434900000` | `1786434900` | 2026-08-11 15:55:00 |

### 2.2 MySQL 时间戳转换函数

```sql
-- 方法1: FROM_UNIXTIME(秒级时间戳)
-- 毫秒 → 秒 → 格式化
SELECT
    ts                                        AS ts_ms,
    FROM_UNIXTIME(ts / 1000)                  AS datetime_raw,
    FROM_UNIXTIME(ts / 1000, '%Y-%m-%d')      AS date_only,
    FROM_UNIXTIME(ts / 1000, '%Y-%m-%d %H:%i') AS datetime_minute,
    FROM_UNIXTIME(ts / 1000, '%H')            AS hour_only
FROM disk_tsar
LIMIT 5;
```

### 2.3 查看数据时间范围

```sql
-- disk_tsar 时间跨度
SELECT
    FROM_UNIXTIME(MIN(ts)/1000) AS earliest,
    FROM_UNIXTIME(MAX(ts)/1000) AS latest,
    DATEDIFF(FROM_UNIXTIME(MAX(ts)/1000), FROM_UNIXTIME(MIN(ts)/1000)) AS span_days
FROM disk_tsar;
-- 结果: 2026-07-01 ~ 2026-08-11，约 41 天

-- pref_tsar 时间跨度
SELECT
    FROM_UNIXTIME(MIN(ts)/1000) AS earliest,
    FROM_UNIXTIME(MAX(ts)/1000) AS latest,
    DATEDIFF(FROM_UNIXTIME(MAX(ts)/1000), FROM_UNIXTIME(MIN(ts)/1000)) AS span_days
FROM pref_tsar;
-- 结果: 2026-07-01 ~ 2026-07-07，约 7 天
```

## 📊 三、按小时汇总指标

### 3.1 核心思路

1. 用 `FROM_UNIXTIME(ts/1000, '%Y-%m-%d %H:00')` 把毫秒时间戳截断到**整点小时**
2. 按 `hostid + mod + 小时` 分组
3. 计算 `AVG`（均值）、`MAX`（最大值）、`COUNT`（该小时采样数）

### 3.2 每小时汇总 SQL

```sql
-- 磁盘指标按小时汇总
SELECT
    hostid,
    mod,
    FROM_UNIXTIME(ts/1000, '%Y-%m-%d %H:00') AS hour_slot,
    COUNT(*)                                   AS sample_count,   -- 该小时采样数
    ROUND(AVG(value), 2)                       AS avg_value,      -- 平均值
    ROUND(MAX(value), 2)                       AS max_value,      -- 最大值
    ROUND(MIN(value), 2)                       AS min_value       -- 最小值
FROM disk_tsar
GROUP BY hostid, mod, hour_slot
ORDER BY hostid, mod, hour_slot
LIMIT 20;
```

### 3.3 性能指标按小时汇总（含主机名、指标描述）

```sql
-- pref_tsar 按小时汇总 + 关联维度表获取可读信息
SELECT
    h.hostname,
    h.location1,
    m.desc                                       AS metric_desc,
    m.unit,
    FROM_UNIXTIME(p.ts/1000, '%Y-%m-%d %H:00')   AS hour_slot,
    COUNT(*)                                      AS samples,
    ROUND(AVG(p.value), 2)                        AS avg_val,
    ROUND(MAX(p.value), 2)                        AS max_val
FROM pref_tsar p
JOIN host_detail h ON p.hostid = h.hostid
JOIN mod_detail m  ON p.mod = m.mod
GROUP BY h.hostname, h.location1, m.desc, m.unit, hour_slot
ORDER BY hour_slot, h.hostname, m.desc
LIMIT 20;
```

### 3.4 CPU 使用率趋势（单台机器某小时）

```sql
-- 查看 host001 的 CPU 指标按小时变化
SELECT
    FROM_UNIXTIME(ts/1000, '%Y-%m-%d %H:00') AS hour_slot,
    mod,
    ROUND(AVG(value), 2)                      AS avg_val,
    COUNT(*)                                   AS samples
FROM pref_tsar
WHERE hostid = 'host001'
  AND mod IN ('cpu_user', 'cpu_sys', 'cpu_idle', 'cpu_wait')
GROUP BY hour_slot, mod
ORDER BY hour_slot, mod
LIMIT 50;
```

## 📊 四、将汇总结果写入新表（可选，用于大屏展示）

```sql
-- 创建小时级汇总表
CREATE TABLE pref_tsar_hourly AS
SELECT
    p.hostid,
    p.mod,
    FROM_UNIXTIME(p.ts/1000, '%Y-%m-%d %H:00') AS hour_slot,
    COUNT(*)                                     AS sample_cnt,
    ROUND(AVG(p.value), 4)                       AS avg_value,
    ROUND(MAX(p.value), 4)                       AS max_value,
    ROUND(MIN(p.value), 4)                       AS min_value
FROM pref_tsar p
GROUP BY p.hostid, p.mod, hour_slot;

-- 添加索引
ALTER TABLE pref_tsar_hourly ADD INDEX idx_host_mod (hostid, mod);
```
